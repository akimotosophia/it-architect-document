# 一覧系設計書のCI

## ビジネスの影響
- Excelの一覧系設計書はオンライン編集などできる一方で、更新履歴が分からなかったり、抜け漏れのある項目や壊れた数式や制御ルールがあって信頼できないことがある。
- Markdownのcsvであれば、更新履歴やGitHub Actionsによるチェックも可能。
- チームでの更新でコンフリクトを防ぐ仕組みは必要だが、設計書のトレーサビリティや品質向上のためにはMarkdown化とGitHub管理はAI時代における効率化の前提である。
